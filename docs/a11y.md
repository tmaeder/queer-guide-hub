# Accessibility conventions

## Card thumbnail alt text (WCAG 1.1.1)

Venue / event / marketplace / news / personality cards on listing pages and
the homepage follow a single rule:

- **If the entity name is rendered as adjacent text inside the same `<a>` or
  `<button>` that wraps the thumbnail**, the thumbnail is decorative — set
  `alt=""`. The link's accessible name already comes from the visible text.
- **If the thumbnail is the only identifier of the entity** (no adjacent
  name, or the name is hidden behind hover/tooltip only), the thumbnail
  carries the entity name: `alt="<entity name> photo"`.

The homepage Trending venues and Upcoming Events Near You sections render
the venue / event name as visible text inside each card link — thumbnails
there are decorative and use `alt=""`.

When in doubt, check the rendered card with a screen reader (or the
browser's a11y inspector): the link should announce the entity name
exactly once.

## Avatar / profile image alt (WCAG 1.1.1 + privacy)

Avatar `<img>` elements always use `alt=""`. The surrounding control
(button, link) carries the user-identifying label via `aria-label`.
**Never put the user's email, real name, or other PII into `alt` text** —
it would leak into the a11y tree, page scrape, and any HTML-to-text
export of the page.
