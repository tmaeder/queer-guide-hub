# design-sync notes — Queer Guide

Repo-specific gotchas for syncing `src/components/ui/` to claude.ai/design.

- **App repo, not a library** — no `dist/`, converter runs synth-entry from `srcDir: src/components/ui`. All 54 canonical components are pinned explicitly in `componentSrcMap` (this skips fuzzy derive AND keeps shadcn subcomponents like `CardHeader`/`DialogTrigger` out of the card list — they still ship in the bundle via `export *`).
- **Toaster excluded**: `sonner.tsx` and `toaster.tsx` both export `Toaster` → ambiguous star export, esbuild drops it from the bundle. Toasts need runtime triggering anyway. `toast`/`useToast` (lowercase) still in bundle.
- **Stale sibling docs**: all 44 `src/components/ui/*.md` files carry false "MUI 7 wrapper" boilerplate (code is Radix/cva). `docsMap` nulls every literal-match component (single-word names match case-insensitively on APFS). Kebab-named docs (`alert-dialog.md` etc.) don't sibling-match PascalCase, no null needed. If the team ever rewrites those docs, remove the nulls.
- **CSS is 100% Tailwind v4 utilities** — components ship no CSS. `cssEntry` = compiled output of `buildCmd` (`npx @tailwindcss/cli@4 -i src/index.css -o .design-sync/.cache/tailwind-ds.css`). MUST re-run after authoring/changing previews so preview-only utility classes are generated (CLI auto-scans repo sources incl. committed `.design-sync/previews/`; gitignored dirs are NOT scanned).
- **Fonts**: Inter only, self-hosted. `src/index.css` `@font-face` uses absolute `/fonts/inter/...` URLs that don't resolve from the bundle → `extraFonts: [".design-sync/fonts-inter.css"]` mirrors them with file-relative paths into `public/fonts/inter/`.
- **App-coupled components** (render-time, not bundle-breaking): `ImageUpload`/`LocationAutocomplete` import the Supabase client (hardcoded URL+anon key, init-safe); `FavoriteButton`/`CountryAutocomplete`/`ProfessionAutocomplete`/`VenueCombobox` use react-query/Supabase hooks → likely need provider or static-prop previews or skip. `sonner.tsx` reads ThemeProvider context.
- **iCloud**: repo lives in iCloud — if git or reads act weird, `brctl download .git`. Trust writes + exit codes over read-backs.
- **DesignSync tool needs claude.ai login** — CLAUDE_CODE_OAUTH_TOKEN sessions can't add design scopes; user must `/login` before project create/upload.
- **Entry anchor trick**: `cfg.entry: "./dist/index.es.js"` points at a nonexistent path ON PURPOSE — soft-fails into synth-entry mode while anchoring PKG_DIR at repo root (the converter walks up from the entry's dirname to the named package.json). Don't "fix" it to a real file.
- **.d.ts pipeline**: synth mode has no type tree → buildCmd chains `tsc --emitDeclarationOnly --noCheck --declarationDir types` + `.design-sync/gen-types-entry.mjs` (writes repo-root `index.d.ts` re-exporting ONLY canonical components from `types/`). `findTypesRoot` picks `types/` automatically. tsc prints TS4058 errors for unrelated admin pages — harmless, ui declarations still emit. Both `types/` and `/index.d.ts` are gitignored.
- **import.meta.env in IIFE**: esbuild empties `import.meta` → non-optional `import.meta.env.X` access THROWS (top-level access killed the whole bundle via `searchClient.ts`). Fixed by optional-chaining (`import.meta.env?.X`) in `src/lib/searchClient.ts` + `src/components/ui/button.tsx`. Any NEW non-optional `import.meta.env.` in the ui import closure re-breaks the bundle — test with the node load smoke (stub window/localStorage, require _vendor react then _ds_bundle.js).
- **Frozen capture clock**: package-capture/validate freeze the page clock → motion/react intro animations and `.img-lazy-fade` never settle → blank-looking renders. `.design-sync/previews/_static.tsx` exports `<StaticState />` (forces `[style*="transform"]` and `.img-lazy-fade` to final state) — render it as a sibling in previews of motion components (EmptyState, MotionCard, StepperShell, TierUpgradeOverlay, AnimatedBeamConnector, ErrorState) and anything using `<Image>`/`<CardImage>`.
- **Images in previews**: app fallback pool is `/images/fallback/*` (public/, unreachable from cards) — always pass an explicit data-URI SVG placeholder (pattern in `previews/Card.tsx`). Never network URLs (capture is effectively offline; picsum flaked).
- **FavoriteButton + ImageUpload are skip-listed** (`cfg.overrides.*.skip`): both call `useAuth()` at render → throw outside AuthProvider; AuthProvider isn't in the bundle (lives outside srcDir). They ship as floor cards. A future `DSProvider` in src/components/ui/ could lift this.
- **Guidelines**: default glob scraped repo ops docs (`docs/*.md`) as "design guidelines" — `guidelinesGlob` pinned to `.design-sync/design-guidelines.md`, authored from CLAUDE.md's Design section. Update it when the design system rules change.
- **Unlayered CSS reset vs @layer utilities (fixed 2026-06-12)**: `src/index.css` had an unlayered `*{border-width:0}` base reset (leftover of the reverted MUI migration, commit 1da4096cf); Tailwind v4 emits utilities inside `@layer utilities`, and an unlayered rule beats ALL layered rules regardless of specificity → every 1px border invisible in the harness bundle AND **in production** (live-site computed-style probe confirmed border-width 0 on .border elements). Fixed by wrapping the reset in `@layer base` — shipped with this sync's commit. Beware grep on minified CSS: `grep -c "@layer"` counts LINES (=1) not occurrences; a wrong all-clear was given on that basis mid-session. Any future unlayered rule in index.css can re-introduce this bug class.
- **BorderFix shims**: wave-forms previews (Input/Textarea/Label/Checkbox/Switch/Slider/Select/FlatFieldGroup/DatePickerWithRange) carry a small `<BorderFix />` style shim from before the @layer fix. Now redundant no-ops (values identical to the utilities) — safe to delete on a later pass.
- **Preview-only Tailwind classes**: the cssEntry is compiled by buildCmd; classes used ONLY in previews exist after a recompile because Tailwind scans the committed `.design-sync/previews/` (gitignored dirs are skipped). ALWAYS re-run buildCmd + full package-build after authoring previews, or new classes silently no-op (waves used inline styles / existing classes as workaround).
- **Known prod bug (flagged, not masked)**: the WCAG tap-target rule (`src/index.css` ~line 444, min-height:44px on buttons) stretches Radix Checkbox/Switch — visible in sheets and in prod. Background-task chip filed by wave-forms.
- **Pagination active-page fix (2026-06-12)**: `PaginationLink` used invalid `backgroundColor: var(--background, #fff)` (HSL-triple var) → active page invisible; source-fixed to `hsl(var(--muted))`.
- **Progress track fix (2026-06-12)**: track was `bg-secondary` (--secondary = 0 0% 4% in this repo, same near-black as the indicator) → solid black bar at any value, in prod too; source-fixed to `bg-muted`.
- **Overlay capture patterns (wave-overlays)**: Radix portals land INSIDE cardMode:single cards (the fixed viewport is the page); popper overlays (Popover/HoverCard/Tooltip/DropdownMenu) need an explicit spacer wrapper so the popper has room; per-preview `<Settle />` (animation/transition none) needed or Radix fade-in-0 holds overlays at opacity 0 under the frozen clock; Radix auto-focus rings in sheets are the real DS focus treatment, not a defect. ContentWarningBanner inherits an oversized overlay viewport — cosmetic; tighten on a future pass.
- **Tap-target rule distorts compact controls (open prod bug, chip filed)**: `src/index.css` `min-height:44px` on button/input also hits Radix Checkbox (16×44 pill) and Switch (~44px egg) — in prod too. Sheets show it faithfully. Fix belongs in app CSS (scope the rule), tracked by the wave-forms chip.
- **Subagent stall pattern**: two overlay agents ended their turn "waiting" for a backgrounded rebuild instead of running it synchronously — instruct wave agents to run build/capture commands SYNCHRONOUSLY in future syncs.

## Re-sync risks (watch-list for the next run)

- **Synth-entry coupling**: the bundle is `export *` over EVERY `.tsx` in `src/components/ui/` — a new file there auto-joins the bundle (add it to `componentSrcMap` + `index.d.ts` via gen script to give it a card); a new top-level `import.meta.env.X` (non-optional) anywhere in the ui import closure KILLS the whole bundle at load (smoke-test: node require _vendor/react then _ds_bundle.js with window/localStorage stubs).
- **Toaster stays ambiguous** until `toaster.tsx` (legacy) or `sonner.tsx` is removed — both export `Toaster`, esbuild drops the symbol.
- **CSS must be recompiled via buildCmd before package-build** whenever src styles OR previews changed — `preview-rebuild.mjs` does NOT refresh `_ds_bundle.css`; stale CSS reproduces the "invisible borders / missing preview classes" class of failure.
- **BorderFix/Settle shims in previews** are tied to current behavior: if `_static.tsx` selectors or the app's animation classes change, sheets may silently regress — regrade after any motion/CSS change.
- **tsc declaration emit prints TS4058 errors** for admin pages — currently harmless (ui d.ts still emit). If ui-relevant declarations ever stop emitting, `.d.ts` bodies silently degrade to `[key: string]: unknown` — spot-check Button.d.ts after build.
- **Playwright cache vanished once mid-session** (~/Library/Caches/ms-playwright) — if validate prints "Executable doesn't exist", `npx playwright install chromium`.
- **Verified-by-upload anchor** lives in the project's `_ds_sync.json` — never upload it over a half-applied write set.
- **2 floor cards are the standing authoring offer**: FavoriteButton, ImageUpload (need an in-bundle AuthProvider/DSProvider to ever preview).
